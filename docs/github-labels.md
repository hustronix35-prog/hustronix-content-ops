# Recommended GitHub Labels

Apply these labels in **Settings → Labels** for issue triage and portfolio polish.

| Label | Color | Description |
|-------|-------|-------------|
| `bug` | `#d73a4a` | Something isn't working |
| `enhancement` | `#a2eeef` | New feature or request |
| `documentation` | `#0075ca` | README, architecture, product docs |
| `good first issue` | `#7057ff` | Good for new contributors |
| `help wanted` | `#008672` | Extra attention needed |
| `voice-quality` | `#fbca04` | Founder Voice rules, post generation |
| `carousel` | `#5319e7` | Design system, Playwright render |
| `slack` | `#1d76db` | Slack workflow, upload, chat agent |
| `linkedin` | `#0e8a16` | LinkedIn API, publish flow |
| `automation` | `#c5def5` | Cursor Automations, cloud agents |
| `security` | `#b60205` | Secrets, auth, vulnerability |
| `tech-debt` | `#fef2c0` | Refactors, legacy cleanup |
| `portfolio` | `#e99695` | Case study, resume, recruiter-facing |
| `priority: high` | `#b60205` | Blocks daily workflow |
| `priority: low` | `#ededed` | Nice to have |
| `wontfix` | `#ffffff` | Not planned |
| `duplicate` | `#cfd3d7` | Duplicate issue |
